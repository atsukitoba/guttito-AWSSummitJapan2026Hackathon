## Must ストーリー実現の Unit 依存

MVP で必須の Must 5本を、どの Unit の組み合わせが実現するか:

```
[ Must 5本 ]
  ├─ 1.1 ロック画面1タップ ───────────────> U1
  ├─ 2.1 AIから話しかけてくれる ──────────> U1 + U2 + U3（全Unit協調）
  ├─ 2.2 整理されていない愚痴を受け止める ─> U2
  ├─ 2.3 沈黙時の相槌/話題提供 ────────────> U2 + U3
  └─ 4.1 複数データソース統合 ────────────> U3
```

**結論**: Must 5本を実現するには **U1 + U2 + U3** の 3 Unit すべてが必須。**U4 は Could のみなので MVP から外せる**。

## 各 Unit の受け入れ基準サマリ

### U1 入口 Unit

**MVP 完成条件**:
- Story 1.1 の全シナリオが iOS TestFlight で動作
- ロック画面ウィジェット → 通話画面起動が実機で確認できる
- NFR-Performance-2（3秒以内）を実測達成

### U2 会話 Unit

**MVP 完成条件**:
- Story 2.1, 2.2, 2.3 の全シナリオが動作
- ペルソナ切替（仮名: まなみ、げんぞう、+1、本番実装時に確定）で応答の違いが確認できる
- 整理強要ゼロの応答が再現性高く得られる（10回中9回以上）
- NFR-Performance-1（MVP 1.5秒）を実測計測

### U3 予習 Unit

**MVP 完成条件**:
- Story 4.1 の全シナリオが動作
- Googleカレンダー・Slack・Gmail の OAuth 連携が動作
- 予習情報が実際のプロンプトに注入されている（ログで確認）
- 通話冒頭の挨拶に予習情報が反映される（Story 2.1 との連携）

### U4 整理・配信 Unit（決勝フェーズ）

**MVP（予選）完成条件**: モック演出でOK

**決勝完成条件**:
- 会話終了後、Bedrock 分析結果が Slack 等に配信される
- マトリクス分類（仕事/プライベート × 報連相/タスク/願望）が正しく動く
- 翌朝リマインダーが配信される

## Story/Unit の健全性チェック

| チェック項目 | 結果 |
|------------|-----|
| 全ての Must ストーリーがどれかの Unit に割り当てられているか | ✅ 5/5 カバー |
| 全てのストーリーがどれかの Unit に割り当てられているか | ✅ 15/15 カバー |
| ペルソナ（めぐみ）が各ストーリーで一貫しているか | ✅ 一貫 |
| 3本柱（R-Accept / R-Entry / R-YouKnowMe）が Unit に分散しているか | ✅ U1=⚡, U2=🫂, U3=🧠, U4=🫂（裏） |
| MVP 必須 Unit の数（U1/U2/U3=3）がチーム人数（3）と一致するか | ✅ 完全一致 |
