## Clarification: S4 への補足

BDD (Behavior-Driven Development) の Given/When/Then 形式を補足します:

- **Given（前提）**: 〜の状況で
- **When（動作）**: 〜をしたとき
- **Then（期待）**: 〜という結果になる

### 例（R-Entry-1 ベース）
```
シナリオ: ロック画面から1タップで起動
Given: スマホがロック状態で、ロック画面にgutittoウィジェットが配置されている
When: ユーザーがウィジェットをタップする
Then: 画面ロック解除せず、500ms以内に音声セッションが開始される
  And: ペルソナから先に挨拶が始まる
```

### 各形式の比較

| 形式 | 特徴 | ハッカソンでの適性 |
|-----|------|--------------|
| A) Given/When/Then | 具体シナリオで体験が伝わる | ★★★★★ (★AI推薦) |
| B) EARS | requirements.md と統一、技術仕様的 | ★★★☆☆ |
| C) 簡素箇条書き | 軽量 | ★★★☆☆ |
| D) 詳細＋数値 | 過剰 | ★★☆☆☆ |
