# Personas

gutittoが届けるべきユーザー像、届けなくてよいユーザー像を定義する。

ペルソナの切り口は「年齢・職業」ではなく、**気持ちや思考の整理スタイル**。この軸で切ることで、gutittoの価値が誰に最も響くかが明確になる。

> **Note**: 以下のペルソナ名（佐藤めぐみ、田中隆一）は**仮の人物像**です。プロダクト設計・プレゼン訴求のための具体化としての仮称であり、実在の個人を指すものではありません。

---

## P1: プライマリペルソナ — 「話して整理する人」

> **キーメッセージ**: 文章にする労力も気力もない。声に出して、誰かに聞いてもらえたら、整理なんていらないままスッキリできる。

### 基本情報

| 項目 | 内容 |
|------|------|
| 名前（仮） | 佐藤 めぐみ（さとう めぐみ） |
| 年齢 | 28歳（20代〜30代のレンジ代表） |
| 職業 | IT企業の企画職（リモート中心、週1〜2出社） |
| 居住 | 東京都目黒区、一人暮らし3年目 |
| 家族 | 両親・姉（既婚）、週に1回程度実家と連絡 |

### 気持ちや思考の整理スタイル

- **文字化が苦手** — 日記を書こうとしても、キーボードに向かうと「書くほどのことか？」と手が止まる。思考が速すぎて、文字に起こすスピードに付いていけない
- **音声派** — 友達と電話すると自然と頭が整理される。「話しながら気づく」タイプ
- **感情先行** — まず感情を吐き出さないと論理が動かない。「で、何が言いたいのか」を先に求められると黙ってしまう
- **疲れた夜に整理できない** — 日中は気を張ってタスクをこなすが、夜になるとエネルギーが尽きて、何も決めたくない
- **完璧主義気味** — 中途半端に書き始めるくらいなら書かない。結果、何も残らない

### 困りごと

- 仕事の報連相が億劫で後回しになる。翌朝慌てて書く → 細部を忘れている
- 友達を呼び出すほどでもない小さな愚痴が溜まる。結局、誰にも話さず寝る
- 気になる人物（上司・同僚）についてのモヤモヤが残ったまま次の日を迎える
- 「今日何があったか」を振り返ろうとしても、ぼんやりとしか思い出せない
- 気がつけば週末、1週間が飛んでいる感覚

### ニーズ

- 整理しなくていい話し相手が欲しい
- 誰にも気を遣わずに愚痴を吐き出したい
- でも、翌朝仕事はスムーズに始めたい（日報や予定の準備は自動で欲しい）
- 私を知っていてくれる感が欲しい（最初から全部説明したくない）
- 画面を操作する気力ゼロの夜でも使える手軽さが欲しい

### 典型的な1日

| 時刻 | 行動 | 気持ち |
|------|------|-------|
| 7:30 | 起床、スマホで Slack / メール確認 | 朝から憂鬱 |
| 9:00 | リモートで業務開始、会議3件 | 集中モード |
| 12:00 | 1人ランチ、SNS眺めつつ |  |
| 13:00 | 午後の会議、資料作成 | 疲れ始める |
| 18:00 | 業務終了、夕食準備 | 解放感半分、ぐったり半分 |
| 20:00 | お風呂、ベッドでゴロゴロ | 脳が止まっている |
| **22:00** | **Netflix見ながら、なんとなくモヤモヤ**。今日の会議で言い切れなかったこと、上司の態度、妹への誕生日プレゼント、色々浮かんでは消える。でも整理する気力はない | **gutitto の出番** |
| 23:30 | 就寝 | 整理しきれず少し不安 |

### ツール利用

- **毎日使う**: Slack、Gmail、Google Calendar、Netflix、LINE、Notion（読み専）
- **週1**: Instagram、X（Twitter）
- **苦手**: 日記アプリ、ToDoリストアプリ（続かない）、タスク管理ツール（Trello等）

### 性格

- 他人の気持ちに敏感、共感力高い
- 自分の気持ちは吐き出すまで気づかないことが多い
- 決断を後回しにしがち
- 完璧主義の傾向、中途半端が嫌、簡単には社会性という鎧を剥がせない（だめになれない）
- 口に出して「あ、そうか」と気づくタイプ

### ゴール / 目標

- 短期: 日々のモヤモヤを翌日に持ち越さない
- 中期: 仕事の振り返りと次の日の準備を、負担なく回せるようになる
- 長期: 自分の気持ちと仕事を両立して、無理なく続けられる生き方

### 引用される言葉（プレゼンで使える）

- 「書くの面倒くさい」
- 「誰かに聞いてほしいけど、呼び出すほどじゃない」
- 「整理してから話せって言われると黙っちゃう」
- 「昔からの友達に電話してるみたいに話したい」

### gutitto が解決する瞬間

仕事終わりの22時、ベッドでゴロゴロしながら、ロック画面のウィジェットをタップ。画面ロック解除も不要。「もしもし、めぐみ？今日お疲れさま。例の資料、無事送れた？」— 数秒前まで整理できていなかったモヤモヤを、相手から聞いてくれる。整理は翌朝、gutittoが Slack に投稿してくれる。めぐみは「話して寝る」だけで、一日が締まる。

## N1: ネガティブペルソナ — 「書いて整理する人」

> **キーメッセージ**: gutitto は 彼女/彼 のための道具ではない。でも、彼女/彼 の存在は gutitto のスコープを明確にしてくれる。

### 基本情報

| 項目 | 内容 |
|------|------|
| 名前（仮） | 田中 隆一（たなか りゅういち） |
| 年齢 | 32歳 |
| 職業 | コンサルティングファーム マネージャー |
| 居住 | 東京都港区、パートナーと同居 |

### 気持ちや思考の整理スタイル

- **文字化が得意** — 日記・ジャーナリング歴5年以上。Obsidian で自分の思考を構造化
- **論理先行** — まず構造化してから感情を扱う。「何が問題か」「選択肢は何か」から入る
- **対話不要** — 1人で整理する方が早い。雑談は気分転換に使うが、整理ツールとしては求めていない
- **ルーティン強し** — 毎朝5:30起床で朝ジャーナル、毎晩22:00に1日の振り返りを書く

### 困りごと（※ gutitto が解決する必要なし）

- 会議が多すぎて自分の作業時間が取れない
- チームメンバーの育成課題
- （gutittoに相談するタイプではない）

### なぜ gutitto が合わないか

- 「整理強要ゼロ」を価値とする gutitto は、構造化を自分でしたい 隆一 にとっては物足りない
- 音声入力より文字入力の方が速い（自分で整理したスピードで思考できる）
- 感情の吐き出しを必要としていない（内省と構造化で完結する）
- AI に感情的に依存するタイプではない（ツールは道具と明確に切り分ける）
- 「受け止める力」を必要としていない

### この存在が示唆すること（スコープ制限）

- gutitto は **全員向けではない** ことを明言すべき
- 「整理が得意な人」には Notion AI / Obsidian / タスク管理ツールが適切
- gutitto のコア価値は **整理しないまま気持ちを吐き出したい人** に届ける
- これにより、プロダクトの訴求ポイントが明確化し、機能追加時にも「隆一向けの機能は入れない」という意思決定ができる

## ペルソナ切り口の設計意図

**伝統的な切り口（年齢・職業・役職）ではなく、気持ちや思考の整理スタイルで切った理由**:

1. **真のターゲットを捉える** — 年齢・職業が同じでも、整理スタイルが違えばニーズは全く違う
2. **スコープを締める** — 「若手 vs ミドル」の切り方だと全員がターゲットに見えてしまう。「話す派 vs 書く派」で明確に分ける
3. **プロダクト哲学と整合** — gutitto は「整理しなくていい存在」。これは整理スタイルを軸にペルソナを定義することと一貫する
4. **プレゼンでの訴求** — 「私たちは、書いて整理できる人のためには作っていません」と言い切れることが、プロダクト設計の覚悟を示す
